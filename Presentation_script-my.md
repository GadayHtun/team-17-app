### 1. HOOK — ပြဿနာ (၃၀ စက္ကန့်)

**[Slide: Title & The Hiring Bottleneck]**
"အခုခေတ် Recruitment အပိုင်းမှာ အကြီးမားဆုံး Bottleneck က ရာနဲ့ချီတဲ့ Technical Applications တွေကို Screening လုပ်ရတာပါပဲ။ ရိုးရိုး Static Quizzes တွေကလည်း Cheat လုပ်ရလွယ်သလို၊ လူကိုယ်တိုင် လိုက်စစ်ဖို့ကျတော့လည်း အချိန်တွေ အများကြီး ကုန်ပါတယ်။ သမရိုးကျ စာမေးပွဲတွေက Senior သမားတွေအတွက် အရမ်းလွယ်ပြီး Junior တွေအတွက်ကျတော့လည်း ခက်ခဲလွန်းနေတတ်တာကြောင့် Candidate ရဲ့ တကယ့် Skill Ceiling ကို တိတိကျကျ တိုင်းတာဖို့ မဖြစ်နိုင်ပါဘူး။"

---

### 2. SOLUTION — ဖြေရှင်းနည်း (၄၅ စက္ကန့်)

**[Slide: Dynamic Evaluation Gist & The Intelligent Filter]**
"ဒါကြောင့် ကျွန်တော်တို့ Team 17 ကနေ LLM-powered **Computer Adaptive Testing (CAT) Engine** ကို အသုံးပြုထားတဲ့ Adaptive Interview Assessment Platform ကို ဖန်တီးခဲ့တာ ဖြစ်ပါတယ်။ ကျွန်တော်တို့ Tool က HR နဲ့ Technical Recruiters တွေအတွက် ဖြစ်ပါတယ်။ Recruitment သမားတွေအနေနဲ့ မတူညီတဲ့ ရာထူးတွေအတွက် စာမေးပွဲမေးခွန်းတွေကို Manual လိုက်ထုတ်နေစရာ မလိုတော့ပါဘူး။ စနစ်ထဲကို Unstructured Job Description (JD) တစ်ခု ထည့်လိုက်ရုံနဲ့ AI က သက်ဆိုင်ရာ Engineering Metrics တွေ၊ Architectural Bound တွေကို အလိုအလျောက် Parse လုပ်ပြီး၊ အဆင့် ၃ ဆင့် (Easy, Medium, Hard) ခွဲခြားထားတဲ့ မေးခွန်းတွေကို On-the-fly ထုတ်ပေးသွားမှာ ဖြစ်ပါတယ်။"

---

### 3. LIVE DEMO — အသုံးပြုပုံ (၃ မိနစ်ခွဲ)

#### **Step 1 — Exam ဖန်တီးခြင်း (Create Exam)**

**[Screen Action: Go to `/create-exam`, fill Job Title, paste JD, enter candidate email]**
"ပထမဆုံးအနေနဲ့ Recruiters Dashboard ရဲ့ `/create-exam` page မှာ Job Title နဲ့ JD ကို Paste လုပ်ပါမယ်။ ပြီးရင် Candidate Email ထည့်ပြီး Easy, Medium, Hard မေးခွန်းအရေအတွက် သတ်မှတ်ကာ Generate ခလုပ်ကို နှိပ်လိုက်ပါမယ်။"

#### **Step 2 — မေးခွန်းများ ပြန်စစ်ခြင်း (Review & Curate)**

**[Screen Action: Go to `/review`, show questions grouped by tier, toggle answers]**
"Generate လုပ်ပြီးတာနဲ့ `/review` page မှာ AI က JD အပေါ် မူတည်ပြီး ထုတ်ပေးထားတဲ့ မေးခွန်းတွေကို Tier အလိုက် မြင်ရမှာ ဖြစ်ပါတယ်။ HR က အဖြေမှန်တွေကို ကြိုကြည့်နိုင်ပြီး၊ မကြိုက်တဲ့ မေးခွန်းဆိုရင် ဖျက်ပစ်တာ ဒါမှမဟုတ် Regenerate လုပ်တာမျိုး စိတ်ကြိုက် လုပ်နိုင်ပါတယ်။ ပြီးတာနဲ့ စနစ်ကနေ သက်ဆိုင်ရာ Candidate အတွက် Security စိတ်ချရတဲ့ One-time link တစ်ခု ထုတ်ပေးမှာ ဖြစ်ပါတယ်။"

#### **Step 3 — Adaptive Testing Flow (Candidate Side)**

**[Screen Action: Open `/exam/[token]`, answer 1-2 questions, show difficulty shifting]**
"အခု Candidate ဘက်ကနေ စာမေးပွဲဖြေဆိုတဲ့ Flow ကို ကြည့်ရအောင်။ ဒီနေရာမှာ ကျွန်တော်တို့ Platform ရဲ့ ထူးခြားချက်ဖြစ်တဲ့ **Adaptive Test UI** ကို မြင်ရမှာပါ။ Candidate က မေးခွန်းတစ်ခုကို မှန်အောင်ဖြေနိုင်ရင် Engine က ပိုခက်တဲ့ Harder Tier မေးခွန်းကို dynamic ပြောင်းလဲပေးသွားမှာ ဖြစ်ပြီး၊ မှားသွားရင်တော့ စနစ်က Difficulty ကို အလိုအလျောက် လျှော့ချပေးသွားမှာ ဖြစ်ပါတယ်။ စုစုပေါင်း မေးခွန်း ၄၀ ဖြေဆိုရမှာ ဖြစ်ပြီး၊ Security အရ Candidate က Difficulty Tier တွေနဲ့ အဖြေမှန်တွေကို ဘယ်လိုမှ မြင်တွေ့ရမှာ မဟုတ်ပါဘူး။"

#### **Step 4 — Submit & Scoring System**

**[Screen Action: Click Submit, show Thank You page, show Score Calculation Formula]**
"ဖြေဆိုပြီးလို့ Submit လုပ်လိုက်တဲ့အခါ Server-side မှာ အလိုအလျောက် Grading တွက်ချက်ပေးပါတယ်။ အမှတ်ပေးစနစ်မှာ ရိုးရိုး Raw Correct Answers ကိုပဲ ရေတွက်တာ မဟုတ်ပါဘူး။ ကျွန်တော်တို့ သုံးထားတဲ့ **Weighted Performance Metric** Formula ($S = \sum_{i=1}^{N} W_i \times C_i$) အရ ခက်ခဲတဲ့ Tier ကို မှန်အောင် ဖြေနိုင်ရင် Credit Points တွေ Exponential တက်လာမှာဖြစ်ပြီး၊ Beginner Concept တွေကို မှားရင်တော့ Weight လျင်မြန်စွာ ကျဆင်းသွားမှာ ဖြစ်ပါတယ်။ ဒါကြောင့် Candidate ရဲ့ တကယ့် True Capacity ကို အမှားအယွင်းမရှိ ပေါ်လွင်စေပါတယ်။"

#### **Step 5 — Results Dashboard**

**[Screen Action: Go to `/results`, show Competency Breakdown table, click Export CSV]**
"နောက်ဆုံးမှာတော့ HR ဘက်ကနေ `/results` page မှာ Candidate ရဲ့ ရလဒ်ကို Competency Breakdown အနေနဲ့ Easy, Medium, Hard အဆင့်အလိုက် အသေးစိတ် Breakdown matrix နဲ့ မြင်တွေ့ရမှာ ဖြစ်ပြီး CSV အနေနဲ့လည်း Export ထုတ်ယူနိုင်ပါတယ်။"

### 4. TECH HIGHLIGHT — နည်းပညာပိုင်း (၄၅ စက္ကန့်)

**[Slide: Platform Technical Architecture & Project Boundaries]**
"ဒီစနစ်ရဲ့ Architecture ပိုင်းကို ပြောရရင်— Frontend ကို **Next.js (TypeScript)** နဲ့ **Tailwind CSS** သုံးပြီး Multi-state UI တွေကို တည်ဆောက်ထားပါတယ်။ Backend ကိုတော့ **Next.js API Routes** နဲ့ မောင်းနှင်ထားပြီး၊ AI အပိုင်းအတွက် **OpenAI APIs** တွေကို ချိတ်ဆက်ကာ Structured JSON Output ထုတ်ယူထားပါတယ်။ Data Persistence အတွက် **MongoDB** ကို Vercel Production Environment မှာ အသုံးပြုထားပြီး၊ Local Development အတွက်တော့ File System Storage ကို အသုံးပြုပါတယ်။

Prompt Engineering အပိုင်းမှာလည်း JD ကို Untrusted Data အဖြစ် ယူဆပြီး Length Cap တွေ ထားရှိပါတယ်။ JSON Parse, Schema Check နဲ့ Quality Check ဆိုပြီး Three-layer Validation စနစ် ထည့်သွင်းထားကာ Retry ၃ ကြိမ်အထိ ပြန်ကြိုးစားအောင် လုပ်ထားပါတယ်။ အဖွဲ့တွင်း Workflow အနေနဲ့ Claude Code နဲ့ TDD workflow ကို သုံးပြီး စနစ်တကျ ရေးသားထားတာ ဖြစ်ပါတယ်။"

**[Slide: Project Boundaries & Anti-Scope]**
"Project ရဲ့ Anti-Scope အနေနဲ့— အချက်အလက် လုံခြုံရေးအတွက် Static Database ထဲမှာ Question Banks တွေ သိမ်းဆည်းထားခြင်း မရှိသလို၊ အလိုအလျောက် Auto-Hiring ဆုံးဖြတ်ချက် ချတာမျိုး မဟုတ်ဘဲ Recruiter တွေ ဆုံးဖြတ်ရလွယ်အောင် Contextual Metrics တွေကိုသာ ထုတ်ပြန်ပေးတာ ဖြစ်ပါတယ်။ တိုက်ရိုက် Video Streaming နဲ့ Live Proctoring အပိုင်းတွေကိုတော့ Initial Scope ထဲမှာ ထည့်သွင်းထားခြင်း မရှိပါဘူး။"
